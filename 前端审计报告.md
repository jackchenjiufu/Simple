## 🟡 中等问题

### 4. 硬编码 IP 地址

`139.196.185.197:7070` 散落在多个文件中。改服务器地址需要改所有文件。
应统一到 `utils/api.js` 的 `baseUrl`。

### 5. 手写 status-bar + nav-bar 重复

25/28 个页面都手写了同样的代码：
```html
<view class="status-bar" :style="{ height: statusBarHeight + 'px' }"></view>
<view class="nav-bar">...</view>
```

每个页面都有 `statusBarHeight` 的 data 定义和 `getSystemInfoSync()` 调用。

### 6. 手写 goBack() 方法重复

10 个页面各自定义了相同的返回方法：
`goBack() { uni.navigateBack(); }`

### 7. v-for 缺 :key

9 处 v-for 循环没有指定 `:key`，HBuilderX 会报 warning：
- message-detail.vue
- tabbar-1.vue (2处)
- tabbar-3.vue (2处)
- my-articles.vue (2处)
- my-feedback.vue
- user-profile.vue

### 8. 重复登录检查

10+ 处手动调用 `uni.getStorageSync('userInfo')`，应提取为 auth mixin。

### 9. 死 CSS

`card-detail.vue`、`message-detail.vue` 残留 `.nav-bar`、`.nav-back`、`.back-icon` 等未使用的 CSS 样式。

## 🟢 轻微问题

### 10. 文件过大

| 文件 | 行数 | 建议 |
|------|------|------|
| message-detail.vue | 774 | 拆分为组件 |
| tabbar-3.vue | 735 | 拆分发布逻辑 |
| user-profile.vue | 652 | 拆分用户卡片组件 |

### 11. 单位混用

同时使用 `upx` 和 `px`，在不同屏幕密度设备上可能出现显示不一致。

### 12. 内联 style

`tabbar-1.vue` 有 `style="height:1px"` 硬编码。

### 13. apiConfig 使用不充分

部分页面直接硬编码 API 地址而非从 `utils/api.js` 导入。
