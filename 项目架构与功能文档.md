# Simple Server 架构与功能文档

## 一、项目概述

**Simple Server** 是一个轻量级 PHP RESTful API 服务框架，采用中间件链 + 服务-仓库分层架构。配套 uni-app（Vue 3）前端和管理后台。

|------|------|

| MySQL 5.7+ | 数据库 |
| PDO | 数据库抽象层 |
| PHPMailer | 邮件发送 |

## 三、架构设计

### 分层架构

```
┌─────────────────────────────────────────┐
│              请求（HTTP Request）         │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│            CORS 中间件                    │
│   设置跨域头，处理 OPTIONS 预检            │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│            日志中间件                     │
│   记录请求体、响应体、处理耗时              │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│           错误中间件                      │
│   try/catch 包裹，统一异常处理             │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│              路由层（Router）             │
│   解析 URI → 分发到对应处理器             │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│            服务层（Service）              │
│   业务逻辑：验证、权限、流程编排           │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│          仓库层（Repository）             │
│   数据访问：SQL 构建、PDO 执行            │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│               MySQL 数据库               │
└─────────────────────────────────────────┘
```

### 中间件链模式

```php
abstract class Middleware {
    protected $next;  // 下一个中间件
    protected $db;    // 数据库连接

    public function setNext(Middleware $next) {
        $this->next = $next;
        return $next;  // 链式调用
    }

    abstract public function handle($request);
}
```

### 两种路由模式

项目包含两套路由系统，可以按需选择：

**模式 A — 直接文件路由**
```
/api/login.php     → 直接访问 PHP 文件
/api/content.php?id=1
/api/admin_users.php
```
- 优点：简单直观，每个文件独立自治
- 适用于：独立部署、快速调试

**模式 B — 集中式路由**（index.php）
```
/api/auth/login    → index.php 解析 URI 后分发
/api/users/1
/api/content
```
- 优点：统一入口，中间件链完整
- 适用于：生产环境

### 文件结构关系

```
server/api/index.php
  ├─ require config/Config.php        # .env 配置读取
  ├─ require config/Database.php      # PDO 连接（含 CORS 预检）
  ├─ require middleware/*.php         # 中间件链
  ├─ require services/UserService.php  # → repositories/UserRepository.php
  └─ require services/ContentService.php # → repositories/ContentRepository.php
```

## 五、核心模块详解

### 1. 配置管理（Config.php）

- 从 `.env` 文件加载配置
- 支持点号路径访问：`Config::get('database.host')`
- 管理：数据库、API、日志、上传、SMTP 五大类配置
- 环境变量与默认值 fallback 机制

### 2. 数据库连接（Database.php）

- 单例 PDO 连接
- 自动从 Config 读取连接参数
- 启用异常模式、关联数组获取、禁用模拟预处理
- UTF-8MB4 字符集

### 3. 中间件系统

| 中间件 | 职责 |
|--------|------|
| CorsMiddleware | 跨域白名单、OPTIONS 预检 |
| LogMiddleware | 请求/响应 JSON 日志、毫秒级计时 |
| ErrorMiddleware | 全局异常捕获、分类处理 |
| AuthMiddleware | Session / Bearer Token 双重验证 |

### 4. 频率限制（RateLimiter）

- 基于 IP + 操作类型
- 数据库持久化计数
- 窗口时间可配置
- 自动清理过期记录
- 示例配置：登录 5分钟/5次，注册 10分钟/3次

### 5. 服务层

#### UserService
- `login()` — 用户名查询 + password_verify
- `register()` — 重名检查 + password_hash + 创建
- `getUserInfo()` / `updateUserInfo()` — 单用户 CRUD
- `changePassword()` — 旧密码验证 + 新哈希
- `getUserList()` — 分页列表

#### ContentService
- `getContentList()` — 多条件筛选 + 分页
- `getContentDetail()` / `createContent()` / `updateContent()` / `deleteContent()`
- `getCarousels()` — 轮播图
- `getRecommendedContent()` — 推荐内容
- 内容所有权验证（update/delete 前检查 user_id）

### 6. 用户画像与分析（user_profile.php）

- **用户画像**：基本信息 + 行为统计（加权） + 偏好标签 + 偏好内容
- **行为权重**：content_click=5, recommend_click=4, page_stay=1
- **内容相似度**：标签 Jaccard 相似度 + 分类匹配 + 标题/正文词汇
- **个性化推荐**：基于偏好标签 → 内容匹配 → 互动+新鲜度排序

### 7. 管理后台

- 管理员登录（Session + is_admin 标识）
- 所有 admin_*.php 文件统一验证 `$_SESSION['is_admin']`
- 功能：用户管理、内容管理、轮播图、消息、日志、统计

## 六、安全机制

| 措施 | 实现 |
|------|------|
| 密码加密 | `password_hash(PASSWORD_DEFAULT)` — bcrypt |
| SQL 注入防护 | 全量 PDO 预处理 + 命名参数绑定 |
| 频率限制 | RateLimiter — IP + 操作类型 + 时间窗口 |
| CORS 控制 | .env 白名单 + 反射 Origin |
| Session 管理 | PHP Session + Bearer Token 双重验证 |
| 文件上传 | 类型白名单 + 大小限制 + 独立上传目录 |
| 错误信息 | 生产环境不应暴露堆栈（待优化） |

## 七、部署指南

### Apache 配置

```apache
<VirtualHost *:80>
    DocumentRoot /var/www/simple
    ServerName your-domain.com

    <Directory /var/www/simple>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
```

### Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/simple;

    location /server/api/ {
        try_files $uri $uri/ /server/api/index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}
```

### 目录权限

```bash
chmod 755 server/uploads/
chmod 755 server/downloads/
chmod 755 server/logs/
chmod 600 server/config/.env
```

## 八、API 接口文档

### 通用约定

- **请求格式**：`Content-Type: application/json`
- **响应格式**：`{"code": 200, "message": "...", "data": {...}}`
- **认证方式**：Session Cookie 或 `Authorization: Bearer <token>`（登录后返回）
- **频率限制**：超限返回 `429 Too Many Requests`

### 认证接口

#### POST /api/login.php — 用户登录
```
请求：{"username": "...", "password": "..."}
成功：{"code": 200, "data": {"user": {...}, "token": "..."}}
失败：{"code": 401, "message": "用户名或密码错误"}
限流：429 — 5分钟内最多 5 次
```

#### POST /api/register.php — 用户注册
```
请求：{"username": "...", "password": "...", "email": "..."}
成功：{"code": 201, "data": {"username": "..."}}
失败：{"code": 400, "message": "用户名已存在"}
限流：429 — 10分钟内最多 3 次
```

### 内容接口

#### GET /api/content.php — 获取内容列表
```
参数：?limit=20&offset=0&status=published
响应：{"code": 200, "data": {"contents": [...], "total": 100}}
```

#### POST /api/content.php — 创建内容
```
请求：{"title": "...", "content": "...", "category": "photography", "tags": "tag1,tag2"}
成功：{"code": 201, "data": {...}}
```

### 关注接口

#### POST /api/follow.php — 关注用户
```
请求：{"user_id": 2}
成功：{"code": 201, "message": "关注成功"}
```

#### DELETE /api/follow.php?user_id=2 — 取消关注
```
成功：{"code": 200, "message": "取消关注成功"}
```

### 上传接口

#### POST /api/upload_image.php — 上传图片
```
Content-Type: multipart/form-data
字段：file（图片文件）
成功：{"code": 200, "data": {"url": "..."}}
限制：jpg/jpeg/png/gif/webp，最大 5MB
```

### 系统接口

#### GET /api/system_monitor.php — 系统监控（需管理员）
```
参数：?type=overview|server|database|api
响应：服务器状态、数据库状态、API 响应时间
```

#### POST /api/deploy.php — 部署文件
```
参数：token + path + content(base64)
验证：token 必须匹配 DEPLOY_TOKEN
```

## 九、常见问题

### Q: 如何重置管理员密码？
访问 `/server/api/reset_admin.php`，重置为 `admin123`

### Q: 如何初始化数据库？
访问 `/server/api/init_database.php` 自动建表

### Q: 文件上传失败？
- 检查 `server/uploads/` 目录权限（需可写）
- 检查 php.ini 中 `upload_max_filesize` 和 `post_max_size`
- 确认文件类型在白名单中

### Q: 如何添加新接口？
在 `server/api/` 下创建 PHP 文件，建议：
1. 引入 `Database.php` 和 `cors_headers.php`
2. 使用 PDO 预处理语句
3. 遵循统一 JSON 响应格式
4. 对敏感操作加入频率限制

### Q: 数据库连接失败？
1. 确认 `server/config/.env` 配置正确
2. 确认 MySQL 服务运行中
3. 检查数据库用户权限
